# 클로드 코드로 비판적 에세이 작성하기

## `/에세이코치` 스킬 활용 가이드

> 2회차(5/29) 실습용 자료
> `/에세이코치` 스킬이 단계별로 안내해주지만, 이 문서를 참고하면 더 좋습니다.

---

## 전체 흐름 (약 2시간)

`/에세이코치`는 5개 블록으로 진행됩니다 (SKILL과 동일):

```
Block 1: 논점 추출 (15분)            ← /에세이코치가 안내
    ↓
Block 2: 에세이 구조 설계 (15분)     ← /에세이코치가 안내
    ↓
Block 3: AI 초안 생성 (15분)         ← /에세이코치가 생성 & 저장 (Sonnet 권장, 1회)
    ↓
Block 4: 비판적 검토 & 수정 (40분)   ← ⭐ 가장 중요! 당신이 주도 (1회 필수, 시간 되면 1회 더)
    ↓
Block 5: 나의 관점 + 최종 저장 (35분) ← essay_final.md 저장 → /심사위원으로 자가 점검
```

> Block 5 마무리 후 별도로 **프롬프트 로그(`prompts.md`)** 를 정리합니다 (평가의 'AI 활용 적절성' 30점과 직결 — 본 가이드 부록 참고).

---

## 시작하기

클로드 코드에서 아래를 입력하세요:

```
/에세이코치
```

고전 텍스트는 1회차 시작 시 선택한 것(`자료/내선택.md`에 기록됨)이 자동으로 사용됩니다 — 별도로 지정할 필요 없습니다.

이후 에세이코치가 단계별로 안내합니다. 아래는 각 블록에서 참고할 팁입니다.

---

## 블록별 팁

### Block 1: 논점 추출

에세이코치가 3가지 논점을 제시합니다. 선택할 때:
- 가장 "쉬운" 것 말고, 가장 **"내가 할 말이 많은"** 것을 고르세요
- 선택 이유를 말할 때, **개인적 경험**이 연결되는 주제가 좋습니다

### Block 2: 구조 설계

에세이코치가 제안하는 구조에 반드시 **수정 요청**을 하세요:
- "본론 2에 AI 시대 관점을 추가해줘"
- "결론에서 개인적 경험을 넣을 수 있게 해줘"
- 구조를 그대로 수락하면 AI의 글이 됩니다. **수정해야 내 글**이 됩니다.

### Block 3: AI 초안 생성

`essay_draft.md` 파일이 생성됩니다 (Sonnet 권장 — `/model`로 전환 후 1회만).
- **바로 다음 단계로 넘어가지 마세요** — 초안을 끝까지 읽으세요
- 읽으면서 "이건 아닌데?" 하는 부분에 메모하세요

### Block 4: 비판적 검토 & 수정 (⭐ 핵심)

에세이코치가 체크리스트를 제공합니다. 아래도 함께 확인하세요:

| 체크 | 질문 |
|------|------|
| □ | 논리적 비약이 있는가? (근거 없는 결론) |
| □ | AI가 틀린 정보를 쓰지 않았는가? |
| □ | 너무 표면적인 분석에 그친 부분은? |
| □ | 이 글이 "내 글"처럼 느껴지는가? |
| □ | 빠진 관점이나 추가할 내용은? |

**1회 필수**로 수정하고, 시간이 되면 한 번 더 다듬으세요. 파일 버전(`essay_v2.md` 등)을 분리해 남기면 개선 사이클을 보일 수 있어 평가에 유리합니다.

### Block 5: 나의 관점 + 최종 저장

AI가 절대 쓸 수 없는 것들을 본인이 직접 추가합니다:
- "이 텍스트를 읽으며 나는..." (개인 감상)
- "수업에서 토론할 때..." (수업 경험)
- 나만의 비유나 사례 (독창적 예시)
- "이 에세이에서 다루지 못한..." (한계 인정)

**직접 쓴 문단**을 에세이코치에게 전달하면 적절한 위치에 삽입해줍니다.
이 블록 끝에 `essay_final.md`가 저장됩니다.

다음 단계로 `/심사위원`을 실행하면 4개 기준으로 본인 에세이를 자가 점검할 수 있습니다 (실제 평가는 5인 심사위원이 별도 진행).

---

## 부록: 프롬프트 로그 정리 ⭐

**평가의 'AI 활용 적절성'(30점) 항목과 직결되는 작업입니다.** 시간이 부족하면 수업 후 본인이 별도 정리해도 됩니다.

작성 과정에서 사용한 핵심 프롬프트 3~5개를 골라 `prompts.md` 파일에 정리하세요. 단순 복붙이 아니라 **왜 그렇게 설계했는지**를 함께 기록하는 것이 핵심입니다.

```markdown
# 에세이 작성 프롬프트 로그

## 프롬프트 #1 — 논점 추출 단계
**입력 프롬프트:**
> 동굴의비유.txt를 읽고 AI 시대의 인식론 관점에서 논점 3개를 뽑아줘.
> 단, "AI가 그림자인가 실재인가" 같은 추상적 비유보다
> 구체적 사례(예: 알고리즘 추천)와 연결되는 논점으로.

**설계 의도:**
첫 시도에서 너무 추상적인 논점만 나와서, "구체적 사례 연결"이라는
제약을 명시적으로 추가함. 역할 부여(인식론) + 출력 형식(3개) + 제약(구체성).

**결과 평가:**
3개 중 2개는 만족스러웠고, 1개는 여전히 추상적 → 최종 선택은 1번.

---

## 프롬프트 #2 — 비판적 검토 단계
(같은 형식으로 작성)
```

**왜 기록해야 하나?**
- 채점위원은 결과물만으로 "이 학생이 AI를 잘 활용했는지" 판단해야 합니다
- 프롬프트 로그가 없으면, 좋은 결과물이 운인지 실력인지 구분이 안 됩니다
- 프롬프트 개선 사이클(첫 시도 → 수정 → 최종)을 보이는 것이 가장 강력한 증거입니다

---

## 핵심 원칙: "AI는 도구, 생각은 내가"

| 하면 좋은 것 | 하면 안 되는 것 |
|---|---|
| AI 초안을 비판적으로 검토 | AI 초안을 그대로 제출 |
| 내 관점과 경험을 추가 | AI에게 모든 것을 맡기기 |
| 여러 번 수정 사이클 반복 | 한 번에 "완성해줘" 요청 |
| AI의 오류를 찾아 수정 | AI 결과를 무조건 신뢰 |
| 과정을 기록하고 성찰 | 결과물만 제출 |
