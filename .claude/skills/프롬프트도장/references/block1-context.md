# Block 1: 구체성의 힘 (컨텍스트 주입)

## EXPLAIN

🥋 **입문 수련: 같은 AI인데, 질문이 달라지면 결과가 달라집니다.**

### Before (나쁜 프롬프트)

```
플라톤에 대해 알려줘
```

이 프롬프트의 문제:
- 범위가 너무 넓다 (플라톤의 뭘?)
- 목적이 불분명하다 (왜 알고 싶은데?)
- 출력 형식이 없다 (어떤 형태로?)

→ AI는 위키피디아 같은 뻔하고 긴 답변을 내놓는다.

### After (좋은 프롬프트)

```
플라톤의 동굴의 비유에서 '그림자'가 상징하는 것을
현대 SNS의 알고리즘 필터버블과 비교해서 3문장으로 설명해줘
```

이 프롬프트에는 3가지 장치가 있다:

| 장치 | 역할 | 예시 |
|------|------|------|
| 📌 범위 한정 | 전체 → 특정 개념 | "동굴의 비유에서 그림자" |
| 🔗 연결 지점 | 추상 → 구체 | "SNS 알고리즘 필터버블과 비교" |
| 📏 출력 형식 | 분량 지정 | "3문장으로" |

**이것이 '컨텍스트 주입'이다.** AI에게 맥락과 조건을 줄수록 결과의 품질이 올라간다.

## EXECUTE

아래 2가지를 순서대로 실행해보세요:

**1. Before 체험** — 이 프롬프트를 그대로 입력하고 결과를 확인하세요:

```
플라톤에 대해 알려줘
```

**2. After 체험** — 이번엔 이 프롬프트를 입력하세요:

```
플라톤의 동굴의 비유에서 '그림자'가 상징하는 것을
현대 SNS의 알고리즘 필터버블과 비교해서 3문장으로 설명해줘
```

두 결과를 비교해보세요. 차이가 느껴지나요?
