# Block 2: 검증 계획 — AI는 가이드, 검증은 학생이 직접

## EXPLAIN

🔍 **AI는 "단서"를, 인간은 "결론"을**

Block 1에서 검증해야 할 주장 5개를 뽑았습니다. 이제 검증할 차례입니다.

**중요한 원칙**: 검증 자체는 AI가 아닌 **여러분이 직접** 합니다. AI는 검증을 **어떻게 할지 가이드만** 제공합니다.

### 왜 직접 검증해야 하나요?

- AI가 검증해도 결국 AI가 만든 답을 AI가 다시 평가하는 셈
- 진짜 출처를 손으로 클릭해 봐야 신뢰도를 체감할 수 있다
- "AI 시대의 핵심 역량 = 인간이 직접 확인하는 습관"

### AI가 만들 검증 가이드 (각 주장당)

| 항목 | 내용 |
|------|------|
| 검색어 후보 3개 | 구글에 입력할 구체적 검색어 (따옴표 활용 등) |
| 신뢰할 만한 출처 유형 | 예: `.edu`, `.gov`, 학술DB, 정부 통계, 1차 사료 |
| 의심 포인트 | "이 주장에서 가장 의심스러운 부분"을 한 줄로 |

### 신뢰도 등급 (학생이 검증 후 매김)

| 등급 | 의미 |
|------|------|
| 🟢 확인됨 | 신뢰할 수 있는 출처 2곳 이상에서 일치 |
| 🟡 부분적 | 핵심은 맞으나 세부 사항이 다름 |
| 🔴 오류 | 출처에서 반증을 발견 |
| ⚪ 미확인 | 1시간 검색해도 출처를 찾지 못함 |

### AI가 자주 저지르는 오류 패턴 (이걸 먼저 의심하세요)

| 패턴 | 예시 |
|------|------|
| 가짜 통계 | "연구에 따르면 78%가..." → 해당 연구 없음 |
| 유령 인용 | "김철수(2019)에 따르면..." → 논문 검색 불가 |
| 뒤섞인 역사 | 연도·인물·사건 세부사항 혼동 |
| 인과관계 포장 | 상관관계를 인과관계처럼 서술 |

## EXECUTE

Block 1에서 추출한 5개 주장에 대해, AI에게 **검증 계획만** 만들어달라고 요청합니다. **검증 실행은 절대 AI에게 맡기지 마세요.**

```
Block 1에서 추출한 주장 5개에 대해, 각각의 검증 계획을 표로 만들어줘.
각 주장마다:
1. 구글에 입력할 검색어 후보 3개 (따옴표·연산자 활용)
2. 신뢰할 만한 출처 유형 (.edu / .gov / 학술DB / 정부통계 등)
3. 가장 의심스러운 부분 한 줄

웹 리서치는 하지 말고, 검증 계획만 만들어줘.
실제 검증은 내가 직접 브라우저에서 수행할 거야.
```

AI 응답을 받은 후:
1. **브라우저를 열고** 각 검색어를 직접 입력
2. 신뢰할 만한 출처 2곳 이상에서 교차 확인
3. 결과를 아래 표 형식으로 본인이 직접 작성:

```markdown
| 주장 | 신뢰도 | 확인한 출처 (URL) | 결론 |
|------|--------|------------------|------|
| 주장 1 | 🟢/🟡/🔴/⚪ | https://... | 한 줄 |
```

작성한 표를 다시 클로드에 붙여넣으면, AI는 **결과를 정리만** 해줍니다 (재검색 X).

## QUIZ

학생에게 **일반 텍스트로 묻는다** (자유 답변형이라 AskUserQuestion 사용 금지):

> **🔍 검증의 교훈**
>
> 직접 검증하면서 가장 놀라웠던 점은? AI가 어떤 부분을 그럴듯하게 틀렸나요?

학생 답변에 대해 (300자 이내):
- 발견한 오류가 AI의 어떤 특성(생성 메커니즘)에서 비롯되는지 짧게 설명
- 학생이 제안한 확인 방법에 공감
- 아래 교훈으로 마무리:

```
🔍 오늘의 교훈:

AI는 "그럴듯하게 틀리는" 천재입니다.
검증 = 인간의 일. AI에게 위임할 수 없는 영역입니다.

⚠️ 의심 우선순위:
  1순위: 구체적 숫자·통계
  2순위: 인용·출처
  3순위: 역사적 세부사항
  4순위: 인과관계 주장

이 습관이 곧 AI 시대의 핵심 역량입니다.
```
