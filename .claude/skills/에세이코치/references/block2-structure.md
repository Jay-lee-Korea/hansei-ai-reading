# Block 2: 구조 설계

## EXPLAIN

📝 **Step 2: 에세이의 뼈대 — 구조를 설계합시다.**

좋은 에세이는 탄탄한 구조 위에 세워집니다. AI에게 구조를 제안받되, **반드시 본인이 수정**해야 합니다.

### 기본 에세이 구조

```
■ 서론: 문제 제기 + 나의 입장(논점)
■ 본론 1: 저자의 주장 분석
■ 본론 2: 저자에 대한 비판 / 반론
■ 본론 3: 현대적 맥락에서의 재해석
■ 결론: 최종 입장 정리
```

### 왜 수정이 필요한가?

AI가 제안하는 구조는 **교과서적으로 무난**합니다. 하지만 무난하다는 건 **개성이 없다**는 뜻이기도 해요.

- AI는 당신이 어떤 수업을 들었는지 모릅니다
- AI는 당신이 어떤 경험을 했는지 모릅니다
- AI는 당신이 어떤 관점에서 글을 쓰고 싶은지 모릅니다

**구조를 그대로 수락하면 AI의 글이 됩니다.** 최소 1가지 이상 수정해야 "나의 글"이 시작됩니다.

### 수정 아이디어

- 본론 순서를 바꿔보기
- 본론 하나를 빼고 다른 관점으로 교체하기
- 서론에 개인적 경험이나 질문으로 시작하기
- 결론에 열린 질문을 남기기

## EXECUTE

아래 과정을 순서대로 실행해보세요.

**1단계: AI에게 구조 설계 요청**

Block 1에서 선택한 논점을 기반으로, 아래와 같은 프롬프트를 입력하세요:

```
[선택한 논점]을 주제로 비판적 에세이 구조를 설계해줘.
서론-본론1~3-결론 형태로 각 섹션에 들어갈 핵심 내용을 정리해줘.
```

**2단계: 구조를 검토하고 반드시 1가지 이상 수정 요청하세요**

AI가 제안한 구조를 읽고, 아래 관점에서 수정할 부분을 찾아보세요:

- "이 순서가 최선인가?"
- "빠진 관점은 없는가?"
- "내가 강조하고 싶은 부분이 충분히 반영되었는가?"

수정 사항을 AI에게 전달하세요. 예시:

```
본론 2에 AI 시대 관점을 추가하고 싶어.
결론은 단정짓지 않고 열린 질문으로 끝내고 싶어.
```

**구조를 그대로 수락하면 AI의 글이 됩니다.** 반드시 수정 요청을 해주세요.

## QUIZ

```json
AskUserQuestion({
  "questions": [{
    "question": "구조에서 어떤 부분을 수정했나요? 왜 그렇게 바꿨나요?",
    "header": "📝 구조 수정 확인"
  }]
})
```

학생의 답변을 확인한 후:
- 수정한 부분이 에세이의 독창성에 기여하는지 피드백한다
- 수정하지 않았다면 "구조를 그대로 쓰면 AI의 글이에요. 하나만 바꿔보세요"라고 다시 안내한다
- 수정이 적절하면 "좋아요! 이제 이 구조로 초안을 만들어볼까요?"로 마무리한다
