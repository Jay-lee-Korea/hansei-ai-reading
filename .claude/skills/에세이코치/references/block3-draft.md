# Block 3: AI 초안 생성 (Sonnet 권장 · 800자 캡)

## EXPLAIN

📝 **Step 3: AI에게 초안을 맡기되, 맹신하지 마세요.**

> **모델 안내**: 이 블록만 Sonnet 권장입니다. `/model` 명령으로 잠시 Sonnet 4.6으로 전환한 뒤 초안 생성 1회만 진행하고, 끝나면 다시 Haiku 4.5로 돌아오세요.

이 단계에서는 확정된 구조를 바탕으로 AI에게 에세이 초안을 작성하게 합니다. 초안은 `essay_draft.md` 파일로 저장합니다. **분량은 800자 이내** (이전 v1: 2000자) — 길이는 학생 본인의 사고로 채울 몫을 남깁니다.

### 초안 생성 시 주의할 점

- AI가 쓴 초안은 **출발점**이지 **완성본**이 아닙니다
- AI 초안은 대체로 매끄럽지만, **깊이가 부족**하고 **개성이 없습니다**
- "잘 읽히네"라는 느낌이 오히려 위험합니다 — 문제가 안 보이기 때문이죠

### 초안을 대하는 자세

| 나쁜 자세 | 좋은 자세 |
|-----------|-----------|
| "잘 썼네, 이대로 내자" | "어디가 약한지 찾아보자" |
| "AI가 나보다 잘 쓰니까" | "AI는 내 생각을 모르니까" |
| "수정할 게 없어 보여" | "수정할 게 없다면 내가 못 찾는 거야" |

**핵심: 초안을 받으면 바로 다음 단계로 넘어가지 말고, 끝까지 읽으세요.**

## EXECUTE

아래 과정을 순서대로 실행해보세요.

**1단계: AI에게 초안 생성 요청**

Block 2에서 확정한 구조를 기반으로, 아래와 같은 프롬프트를 입력하세요:

```
아래 구조로 비판적 에세이 초안을 작성해줘.
분량은 800자 이내로, 학술적이지만 읽기 쉬운 문체로 써줘.
초안은 출발점일 뿐이니 핵심 논증만 담고 살은 내가 붙일 거야.
작성한 초안을 essay_draft.md 파일로 저장해줘.

[확정된 구조를 여기에 붙여넣기]
```

**2단계: 초안을 끝까지 읽으세요**

저장된 `essay_draft.md` 파일을 열어서 **처음부터 끝까지** 읽어주세요.

읽으면서 아래를 메모하세요:
- "이상한데?" 하는 부분
- "이건 아닌데?" 하는 부분
- "여기 약한데?" 하는 부분
- "내 생각이랑 다른데?" 하는 부분

**바로 다음으로 넘어가지 말고 초안을 끝까지 읽으세요.** 이 과정이 다음 단계(비판적 검토)의 밑거름이 됩니다.

## QUIZ

```json
AskUserQuestion({
  "questions": [{
    "question": "초안을 읽었나요? 어떤 부분이 마음에 안 드나요?",
    "header": "📝 초안 검토 확인"
  }]
})
```

학생의 답변을 확인한 후:
- 마음에 안 드는 부분을 구체적으로 이야기하면 "좋은 눈이에요!"라고 격려한다
- "다 괜찮은데요"라고 하면 "정말요? 결론의 주장이 충분히 강한가요? 본론의 근거가 설득력 있나요?"라고 한번 더 살펴보도록 유도한다
- "다음 단계에서 본격적으로 수정할 거예요. 지금 느낀 불편함을 기억해두세요!"로 마무리한다
