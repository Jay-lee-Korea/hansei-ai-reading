# Block 4: 비판적 검토 & 수정

## EXPLAIN

⭐ **Step 4: 가장 중요한 단계 — 비판적 검토**

이 단계가 에세이의 품질을 결정합니다. AI가 쓴 초안을 그대로 제출하면 "AI의 글"이지만, 비판적으로 검토하고 수정하면 "나의 글"이 됩니다.

### 비판적 검토 체크리스트 5항목

아래 5가지 관점으로 초안을 검토하세요:

```
□ 1. 논리적 비약 — 근거 없이 결론으로 점프하는 부분이 있는가?
□ 2. 사실 오류  — AI가 틀린 정보를 쓰지 않았는가?
□ 3. 깊이 부족  — 표면적 분석에 그치고 깊이가 부족한 부분은?
□ 4. 나의 목소리 — "나의 글"처럼 느껴지는가, "AI의 글"인가?
□ 5. 빠진 관점  — 꼭 넣고 싶은데 빠진 내용은 없는가?
```

> **재료 활용 팁**: `중간과제/탐구일지.md`에 학생이 1회차에 발견했던 **"AI가 틀리거나 부족했던 점"** 메모와 `중간과제/에세이재료.md`의 **"4. 반론·약점"** 섹션이 있으면, 이 둘을 체크리스트와 대조해 학생에게 보여준다. "본인이 이미 발견한 AI의 한계 패턴이 이 초안에서도 보이지 않나요?"라고 물어 비판적 검토를 본인의 누적 경험과 연결시킨다.

### 왜 이 단계가 가장 중요한가?

- AI는 **그럴듯하지만 틀린 말**을 자연스럽게 씁니다
- AI는 **깊이 있는 분석** 대신 **표면적 요약**을 하는 경향이 있습니다
- AI는 **당신의 생각**을 넣을 수 없습니다 — 오직 당신만 할 수 있어요

**"이 문장, 진짜 네 생각이야 아니면 AI 생각이야?"** — 이 질문을 매 문단마다 해보세요.

### 수정 사이클 (1회 필수 + 1회 선택)

- **1차 수정 (필수)**: 체크리스트 기반 큰 틀 수정 — 모든 학생이 반드시 수행
- **2차 수정 (선택)**: 문장 단위 세밀한 수정 — 시간·토큰 여유가 있으면 권장

> v1은 "최소 2회 필수"였으나, v2는 토큰 보호를 위해 "1회 필수 + 1회 선택"으로 완화했습니다. 단, **반드시 1회는** 거쳐야 학생 본인의 글이 됩니다.

## EXECUTE

아래 과정을 순서대로 실행해보세요.

**1단계: 체크리스트 기반 1차 수정 요청**

초안(`essay_draft.md`)을 검토한 결과를 바탕으로, AI에게 수정을 요청하세요. 예시:

```
essay_draft.md를 아래 기준으로 수정해줘:
1. 본론 2의 반론이 약해 — 구체적 사례를 추가해줘
2. 결론이 너무 일반적이야 — 나의 입장을 더 명확하게 해줘
3. 서론의 문제 제기가 뻔해 — 더 도발적인 질문으로 시작해줘

수정한 결과를 essay_v2.md로 저장해줘.
```

**2단계: 2차 수정 (선택)**

`essay_v2.md`를 다시 읽고, 시간·토큰 여유가 있다면 문장 단위로 검토하세요:

- **이 문장, 진짜 네 생각이야 아니면 AI 생각이야?**
- 모든 주장에 **근거**가 있는가?
- **나만의 표현**이 들어가 있는가?

수정한다면 `essay_v3.md`로 저장하세요. 1차 수정만으로 충분히 본인 글이 되었다고 판단하면 다음 블록으로 넘어가도 됩니다.
