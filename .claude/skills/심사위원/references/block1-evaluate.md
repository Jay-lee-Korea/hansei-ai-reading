# Block 1: 4개 기준 평가 — 기준별 자가 점검

## EXPLAIN

⚖️ **자가 점검의 4개 기준**

실제 5인 심사위원이 사용하는 것과 **동일한 기준**으로 본인 에세이를 미리 점검합니다. "좋다/나쁘다"라는 막연한 판단이 아니라, 기준별로 약점을 구체적으로 짚어내는 것이 목표입니다.

### 심사 기준표

| 기준 | 배점 | 핵심 질문 |
|------|------|-----------|
| 📊 비판적 사고의 깊이 | 40% | 저자의 주장에 대해 깊이 있게 생각했는가? |
| 🤖 AI 활용의 적절성 | 30% | AI를 도구로 잘 활용했는가, 아니면 AI에 의존했는가? |
| 🔗 논리 구조와 설득력 | 20% | 주장 → 근거 → 결론의 흐름이 논리적인가? |
| ✨ 독창성 | 10% | 자기만의 관점이나 새로운 통찰이 있는가? |

### AI가 쓴 흔적 탐지 패턴

에세이에서 아래 패턴이 발견되면 AI 의존도가 높을 수 있습니다:

| 패턴 | 예시 | 문제 |
|------|------|------|
| 애매한 표현 반복 | "~라고 볼 수 있다", "~측면에서" | AI 특유의 회피적 어투 |
| 균형잡힌 양비론 | "A도 맞지만 B도 맞다" (결론 없음) | 입장을 정하지 않는 AI 스타일 |
| 경험 부재 | 추상적 서술만, 구체적 경험 없음 | AI는 개인 경험이 없음 |
| 균일한 깊이 | 모든 문단의 수준이 비슷함 | 사람은 관심 분야에서 더 깊게 씀 |

### 평가 출력 형식

각 기준에 대해 아래 형식으로 평가합니다:

```
📊 기준 1: 비판적 사고의 깊이 (40%)
점수: ★★★★☆ (8/10)

✅ 잘한 점: [구체적 인용과 함께]
⚠️ 개선할 점: [구체적으로]
💡 개선 제안: [실행 가능한 제안]
```

## EXECUTE

아래 프롬프트를 실행해보세요:

```
[에세이 파일명]을 읽고 아래 4개 기준으로 상세하게 평가해줘.

1. 비판적 사고의 깊이 (40%)
2. AI 활용의 적절성 (30%) — AI가 쓴 흔적도 찾아줘
3. 논리 구조와 설득력 (20%)
4. 독창성 (10%)

각 기준마다:
- 10점 만점으로 점수
- 잘한 점 (에세이에서 구체적 문장 인용)
- 개선할 점 (구체적으로)
- 개선 제안 (실행 가능하게)
```

> `[에세이 파일명]`을 실제 파일명으로 바꿔서 실행하세요. 파일이 없으면 에세이 텍스트를 직접 붙여넣어도 됩니다.

결과를 읽어보고, 각 기준별 평가가 납득되는지 확인해보세요.
